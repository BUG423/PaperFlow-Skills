# Idea 规范约定 (ideation → experiment 交接契约)

## 存储位置

实验阶段从以下固定位置读取 idea:

```text
paperflow/
├── idea-spec.md       # 选定 idea 详细说明书 (详见模板)
├── references.md      # 结构化相关文献 (详见模板)
└── ideation.md        # 完整 10 候选清单 + 选定记录 (归档用)
```

**必须存在的最小集合**: `idea-spec.md` + `references.md`。缺少任一份 → 实验阶段拒绝启动，提示"先跑完 ideation 落盘"。

## idea-spec.md 必须包含的字段

以下字段是实验阶段的硬依赖——缺一个都会导致后续实验指南不完整:

### 必须字段

| 字段 | 说明 | 示例 |
|------|------|------|
| `## 一句话总结` | ≤30 字 | "TinyIO 的 DPAA 注意力训练多分支→推理合并单路" |
| `## 核心 Claims` | C1 (必填), C2/C3 (可选) | "在 IO 上首次验证注意力可重参数化" |
| `## 关键假设` | H1 (必填), H2 (可选) | "多分支 softmax 可等价/近似合并且 ATE 损失<1%" |
| `## 技术路线` | 整体框架 + 核心模块 + 关键设计选择 | 输入→backbone→关键模块→输出; 分支数/temperature/... |
| `## 和 closest work 的本质区别` | 一句话 delta | "TinyIO 重参数化只到卷积层，本工作推到注意力" |
| `## 实验设计 (初步)` | 主实验 + 消融 + H1 pilot | 至少一个主实验和一个 pilot |
| `## 关键风险 & 退路` | 至少 H1 不成立的退路 | "H1 失败→切 Backup I2 (RepSGU)" |
| `## 预期投稿` | venue + 实验周期 + writing 周期 | "ICRA 2027 / 实验 2.5 月 / writing 4 周" |

### 字段校验规则

- `一句话总结`: 非空，≤30 字
- `核心 Claims.C1`: 非空，含"在 XX 设定下, 通过 XX 方法, 能做到 XX 结果"结构
- `关键假设.H1`: 非空，含可验证的量化判据 (e.g. "ATE 损失<1%" 而非 "效果不差")
- `技术路线`: 含"整体框架"行 + 至少 1 个"核心模块"
- `和 closest work 的本质区别`: 非空，点名 closest work 的具体名称
- `实验设计`: 含至少 1 行主实验 + 1 行 pilot
- `关键风险.退路`: H1 失败退路非空

## references.md 必须包含的字段

| 字段 | 说明 |
|------|------|
| `## 第一层: 直接竞争者` | ≤3 篇, 每篇含: 做了什么 / 差异 delta / 局限 |
| `## 第二层: 同一子方向` | 5-8 篇, 每篇含: 做了什么 / 关系 |
| `## 第三层: 方法来源` | 3-5 篇, 每篇含: 提出了什么 / 本工作怎么用 |
| `## 第四层: IO 领域基础` | 3-5 篇, 至少含 RoNIN + IONet + TLIO |
| `## 检索元信息` | query / 日期 / N 篇 / M 篇用作参考 / 源状态 |

## ideation.md 必须包含的字段 (归档用)

| 字段 | 说明 |
|------|------|
| 用户处境 | 领域/资源/入场状态 |
| 选定方向 | 主推+Backup, 含 claim + H1 + 评级 |
| 完整 10 候选清单 | 表格, 含贡献类型+评级 |
| 检索元信息 | query/时间窗/检索数 |

## 校验流程

实验阶段开始时, 读 `idea-spec.md` + `references.md`, 逐字段校验:

1. 必须字段全部非空 → 进 Phase 1
2. 有必须字段缺失 → 退回, 列出缺失清单, 要求补全后重新进入
3. 字段内容明显不完整 (e.g. H1 无量化判据) → 警告但放行, 提示"建议补全后再实验"

## 和 ideation skill 的关联

ideation skill 的 Step 6 落盘时, 必须严格按此规范输出。实验阶段不做"从对话历史推断 idea"——只读磁盘文件。
