# PaperFlow Skills

一套为 Claude Code 设计的学术论文全流程辅助 skill，覆盖从模糊研究兴趣到投稿准备的完整周期。

## 和一般论文辅助工具有什么不同

大多数论文写作工具把自己定位成"模板引擎"——填好字段，输出文本。PaperFlow 的定位是一个**研究搭档**：会追问你的假设，会帮你搜文献检验新颖性，会逐段和你讨论论文的写法，会在投稿前用审稿人的眼光挑毛病。

核心原则：
- **追问比填表重要**。不说"请输入你的研究问题"，而是"你为什么觉得这个问题重要？现有方法在哪里失败了？"
- **证据比修辞重要**。不帮你写"我们的方法达到了 SOTA 性能"，而是问"领先了多少？在什么数据集上？统计显著吗？"
- **逐段确认，你掌控**。每写一段就等你确认，你可以随时纠正方向。你不是在看 AI 写论文，而是在和 AI 讨论论文。

## Skill 总览

| Skill | 做什么 | 什么时候用 |
|-------|--------|-----------|
| `paper-ideation` | 把模糊兴趣变成可执行的研究方向，含文献验证和严格评审 | 有一个模糊的想法，不确定值不值得做 |
| `paper-experiment` | 系统检索文献 → 提取实验模式 → 设计达到目标 venue 水准的实验方案 | idea 确定了，需要设计实验 |
| `paper-writing` | 逐段交互式写论文，中文初稿确认后翻译为英文 LaTeX | 实验结果都有了，开始动笔 |
| `paper-prereview` | 模拟真实审稿流程，从 desk check 到多 reviewer 评分到 AC 意见 | 初稿完成，投稿前自查 |
| `paper-rebuttal` | 分类审稿意见，制定回应策略，起草具体回复 | 收到审稿意见，需要写 rebuttal |

## 工作流

```
模糊想法
    │
    ▼
paper-ideation          ─── 讨论、追问、搜索、评审
    │                       产出一个"值得做"的研究方向
    ▼
paper-experiment        ─── 搜文献、分析实验模式、确认环境
    │                       产出可执行的实验方案 + 引用列表
    ▼
paper-writing           ─── 逐段写中文初稿 → 确认 → 翻译英文
    │                       产出一篇完整的论文
    ▼
paper-prereview         ─── 模拟审稿，找出投稿前必须改的问题
    │                       产出修改优先级列表
    ▼
投稿
    │
    ▼（收到审稿意见）
paper-rebuttal          ─── 分类意见、制定策略、起草回复
```

## 安装

```bash
# 克隆到 Claude Code 的 skills 目录
git clone <repo-url>
mkdir -p ~/.claude/skills
cp -R paper-* ~/.claude/skills/
```

安装后直接在对话中描述你的需求即可，不需要记命令：

- "我有一个模糊的想法，帮我捋一捋值不值得做"
- "idea 确定了，帮我查文献、设计实验"
- "实验结果都出来了，帮我开始写论文的方法部分"
- "初稿写完了，帮我模拟审稿看看有什么问题"
- "审稿意见回来了，帮我分析一下怎么写 rebuttal"

## 设计哲学

### 研究搭档，不是秘书

每个 skill 被设计为会主动思考和提问，而不是被动等待指令。在 idea 阶段会追问你的假设，在实验阶段会指出文献中的实验模式，在写作阶段会提醒你 claim 和 evidence 的对齐。

### Claude Code 原生

充分利用 Claude Code 的能力：实时文献搜索、文件读写、代码执行、跨会话记忆。不是生成一个静态的模板，而是在真实的项目目录中创建可用的文件。

### 交互优先，但可以加速

默认每一步都和你确认。如果你觉得节奏太慢，可以说"直接给出完整结果"，skill 会切换到自主模式。

### 诚实 > 好听

不编造引用、不伪造结果、不给虚假信心。不确定的事情说"不确定"。这比一个听起来很流畅但充满幻觉的输出有价值得多。

## 边界

这些 skill 不会帮你做实验，不能保证论文被接收，不能替代领域专家的判断。它们帮你做的是：**把思考结构化、把盲区暴露出来、把修改路径具体化**。
